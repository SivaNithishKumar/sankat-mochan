Record 01Command post · voice pipeline

Language identification on FLEURS

Language ID decides which vocabulary IndicConformer decodes an SOS clip with — get it wrong and the transcript is garbage. Replacing the mass-based scorer with a peak-confidence scorer (plus a calibrated Hindustani margin) cut errors from 17 to 2 on 180 clips.

The residual two errors are the Hindi↔Urdu pair — the same spoken language (Hindustani), acoustically inseparable. That is a real limit, not a scorer bug.

Verdict
✓ SHIPPED Sahayak is the default model. Stock-E2B speed (15.6 vs 16.3 tok/s) and size (3.11 GB) with E4B-class field guidance — best quality-per-watt for the emergency use case.
✗ E4B 2× slower (7.0 tok/s), +1.7 GB on disk, ~5.5 GB RAM to load — marginal quality gain doesn’t justify it on a phone.
NOTE All Gemma-4 variants emit a [thinking] block at the CLI; the app sets enable_thinking=false — verify suppression holds in-app.
Record 03Fine-tune · held-out eval

What the fine-tune actually taught the model

Raw Gemma 4 E2B against the Sahayak LoRA on a never-seen held-out set — same system prompt, same greedy decoding, graded 1 / 0.5 / 0 against key facts, format, language, and safety. Accuracy doubled, and every safety-critical failure of the base model was fixed.

Open gaps — flagged for round two
⚠ HUMAN Anaphylaxis (A-0260) fails in both models — neither recognizes throat-tightening + wheezing after stings, neither mentions an adrenaline auto-injector. Needs human review and a targeted training top-up.
⚠ Multilingual barely moved (38% → 43%) — only ~3 training examples per non-English language; the tuned model answers in-language but sometimes degenerates into repetition. Data-volume problem, not an architecture problem.
⚠ Numeric allocation wobbles — the fine-tune even regressed on C-0157, assigning 36 of 18 available volunteers.
→ Next step: data top-up on Marathi / Telugu / Bengali relay packets and anaphylaxis-class first aid, then a round-two fine-tune.
Record 04Snapdragon X Elite · GenieX command post

Command-post model selection & NPU latencies

Head-to-head triage of regional-language disaster SOS messages (Tamil, Telugu, Bengali, Marathi, Malayalam, Hindi) across five candidate models, plus the measured NPU compile of the speech pipeline. Gemma 4 E4B won on both fidelity and speed; Sarvam-M was 30–80× slower and made safety-critical translation errors.

session notes · 2026-07-11 (not yet in a repo doc) GenieX v0.3.14 · OpenAI-compatible serve multilingual triage battery · 6–16 cases
Per-query triage latency — log scale
position encodes seconds per query; verdicts carry the quality findings

Fidelity notes: Gemma E4B — 6/6 faithful, no reasoning block, no refusals. Llama heretic — fast and uncensored but embellishes (“mud”→“debris”) and leaked Hindi into triage JSON. Qwen3-4B — over-triages everything to critical, hallucinates disaster types on Dravidian input. Sarvam-M — reasoning always on; Tamil “roof”→“floor”, Malayalam “landslide”→“flood”, hallucinated rains in Marathi.
